### 2️⃣-A 智能匹配详细流程 (hybrid_search)

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                    智能匹配核心流程 (/api/ai-match → hybrid_search)              │
└─────────────────────────────────────────────────────────────────────────────────┘

用户输入
├── product_name: "注射用交联透明质酸钠凝胶"  (产品名称，用于过滤)
└── requirement: "适用于面部真皮组织 0.8ml/支"  (参数需求，用于向量排序)
            │
            ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         第一阶段：产品名称过滤 (硬性条件)                          │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   product_name 有值?                                                            │
│        │                                                                        │
│        ├── 是 ──► 精确匹配: SELECT di_code WHERE product_name = ?               │
│        │              │                                                         │
│        │              ├── 有结果 → candidate_di_codes = 匹配的 di_code 集合      │
│        │              │                                                         │
│        │              └── 无结果 → 模糊匹配: WHERE product_name LIKE '%xxx%'     │
│        │                              │                                         │
│        │                              ├── 有结果 → candidate_di_codes = 集合    │
│        │                              └── 无结果 → candidate_di_codes = None    │
│        │                                          (全量检索)                    │
│        │                                                                        │
│        └── 否 ──► candidate_di_codes = None (跳过产品名称过滤)                   │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
            │
            ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         第二阶段：参数需求向量检索 (语义排序)                      │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   requirement 有值?                                                             │
│        │                                                                        │
│        ├── 否 ──► 直接返回第一阶段结果 (按 last_updated 排序, matchScore=80)     │
│        │                                                                        │
│        └── 是 ──► 进入向量检索流程                                               │
│                        │                                                        │
│                        ▼                                                        │
│              ┌─────────────────────────────────────────────────────────┐        │
│              │  尝试 FAISS 检索 (embedding_faiss.py)                   │        │
│              │  faiss_search(requirement, top_k=500)                   │        │
│              │                                                         │        │
│              │  FAISS 索引存在?                                         │        │
│              │       │                                                 │        │
│              │       ├── 是 → 向量 ANN 搜索，召回 500 条                │        │
│              │       │        计算精确余弦相似度                        │        │
│              │       │                                                 │        │
│              │       └── 否 → 降级到 vector_search()                   │        │
│              └─────────────────────────────────────────────────────────┘        │
│                        │                                                        │
│                        ▼                                                        │
│              ┌─────────────────────────────────────────────────────────┐        │
│              │  降级: vector_search() (embedding_service.py)           │        │
│              │                                                         │        │
│              │  Stage 1: FTS5 粗召回 (1000条)                          │        │
│              │     • Jieba 分词提取关键词                               │        │
│              │     • FTS5 MATCH 查询 (OR 关系)                         │        │
│              │     • 召回不足时 LIKE 补充                               │        │
│              │                                                         │        │
│              │  Stage 2: 向量重排                                       │        │
│              │     • 调用 Embedding API 获取查询向量                    │        │
│              │     • 从 embeddings 表获取候选产品向量                   │        │
│              │     • 计算余弦相似度，按相似度排序                        │        │
│              │                                                         │        │
│              │  API 失败? → 直接返回 FTS 结果                           │        │
│              └─────────────────────────────────────────────────────────┘        │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
            │
            ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         第三阶段：结果合并与后处理                                │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   candidate_di_codes 有值? (第一阶段产品名称过滤结果)                            │
│        │                                                                        │
│        ├── 是 ──► 交集过滤: 只保留 di_code ∈ candidate_di_codes 的结果          │
│        │              │                                                         │
│        │              └── 过滤后 < 10 条? → 放宽条件，产品名称匹配的排前面        │
│        │                                                                        │
│        └── 否 ──► 保留全部向量检索结果                                           │
│                                                                                 │
│   厂家去重 (dedupe_by_manufacturer=True)                                        │
│        │                                                                        │
│        └── 每个厂家只保留排名最靠前的一条                                         │
│                                                                                 │
│   计算 matchScore                                                               │
│        │                                                                        │
│        └── matchScore = similarity × 100 (直接百分比)                           │
│            例: similarity=0.75 → matchScore=75                                  │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
            │
            ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│   返回结果                                                                       │
│   {                                                                             │
│     "success": true,                                                            │

│       {                                                                         │

│         "product_name": "...",                                                  │
│         "manufacturer": "...",                                                  │
│         "matchScore": 75,        ← 符合率 (0-100)                               │
│         "rank": 1,               ← 排名                                         │
│         "final_score": 0.75      ← 原始相似度                                   │
│       },                                                                        │
│       ...                                                                       │
│     ],                                                                          │
│     "total": 50,                                                                │
│     "method": "vector"           ← 检索方法 (vector/fts)                        │
│   }                                                                             │
```

## 🎯 设计亮点

### 1. "本地湖 + 云端补" 混合架构
- **90% 请求**：本地 SQLite + FTS5，毫秒级响应，零成本
- **10% 请求**：云端 API 补充，保证数据权威性

### 2. 两阶段语义检索 (hybrid_search)
```
┌─────────────────────────────────────────────────────────────────┐
│  第一阶段: 产品名称过滤 (硬性条件)                                │
│  ─────────────────────────────────────────────────────────────  │
│  用户输入产品名称 → 精确/模糊匹配 → 得到候选 di_code 集合         │
│                                                                 │
│  第二阶段: 参数需求向量排序 (语义匹配)                            │
│  ─────────────────────────────────────────────────────────────  │
│  用户输入参数需求 → 向量检索 → 按相似度排序                       │
│                                                                 │
│  合并: 取交集 + 厂家去重 + 计算 matchScore                       │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 多级降级策略
```
优先级 1: FAISS 向量检索 (最快，~50ms)
    ↓ FAISS 索引不存在
优先级 2: FTS5 粗召回 + 向量重排 (~200ms)
    ↓ Embedding API 失败
优先级 3: 纯 FTS5 结果 (无向量排序)
```

### 4. 向量索引优化
- **IVF+PQ 压缩**：255万向量，内存占用 < 500MB
- **批量生成**：阿里云 Batch API，成本降低 50%
- **增量更新**：text_hash 检测变更，避免重复计算

### 5. 智能数据同步
- **自动补全**：检测遗漏日期，批量下载
- **策略选择**：>7天用周包，>30天用月包
- **UPSERT 机制**：增量更新，不重复导入
